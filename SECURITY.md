# Security Policy

## Supported Versions

The project maintains security fixes for the current `main` and `develop` branches.

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability, please do not open a public issue or pull request.

Report the issue privately through GitHub Security Advisories.

When submitting a report, include as much detail as possible:

- Affected URL, file, component, or feature
- Clear steps to reproduce the issue
- Proof of concept, if available
- Expected and actual behavior

The maintainers will acknowledge the report as soon as possible and coordinate remediation privately before any public disclosure.
